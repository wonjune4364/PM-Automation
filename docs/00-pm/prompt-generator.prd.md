> Product: BJI-IT (NextCursor) — Next.js 15 웹 애플리케이션

## 1. 요약 (Executive Summary)

NextCursor는 한국어 개발자, 주니어 PM, 비기술 창업자(바이브 코더)가 AI 코딩 도구에 넣을 기획 문서를 5단계 순차 워크플로우(PRD → TRD → IA → Use Cases → Design)로 10분 안에 생성할 수 있는 무료 프롬프트 생성기 스위트다.

이 PRD는 현재 제품의 최대 기회인 **단계 간 컨텍스트 단절 문제**(Opportunity Score 0.86)를 해결하고, 풀체인 워크플로우 완료율을 90일 내 30%까지 높이기 위한 기능 요건과 출시 전략을 정의한다.

### 가치 전달 요약 (4-Perspective Value Delivered)

| 관점 | 현재 상태 | 개선 후 상태 | 측정 지표 |
|------|---------|------------|---------|
| 사용자 (User) | 생성기 간 출력물을 수동으로 복사·붙여넣기 하며 맥락을 잃음 | "다음 단계로 보내기" 버튼 한 번으로 이전 출력물이 자동 주입됨 | 풀체인 완료율 30% (현재 ~0%) |
| 비즈니스 (Business) | 세션 종료율이 높아 재방문율이 낮음 | 순차 워크플로우가 재방문과 구독 전환의 핵심 레버로 작동 | 주간 활성 프롬프트 생성자(WAPG) +40% |
| 기술 (Technical) | 생성기 간 데이터 모델이 분리되어 컨텍스트 공유 불가 | sessionStorage 기반 경량 컨텍스트 스토어로 상태 전달 | 추가 백엔드 없이 구현 가능 |
| 시장 (Market) | "또 다른 AI 글쓰기 도구"로 포지셔닝이 불명확 | "Cursor를 시작하기 전에 거치는 한국어 기획 워크플로우"로 차별화 | SEO 유입 키워드 순위 상승 |

## 2. 문제 정의 (Problem Statement)

### 핵심 문제

한국어 사용 개발자, PM, 바이브 코더는 Claude나 ChatGPT 같은 AI 도구로 기획 문서를 만들 때 두 가지 근본적인 어려움에 부딪힌다.

첫째, **AI 친화적 프롬프트를 작성할 PM 도메인 지식이 없다.** 빈 페이지에서 PRD를 작성하라는 요청은 기술적 능력과 무관하게 사람을 멈추게 한다.

둘째, **단계 간 연결이 없어 문서 일관성을 유지하기 어렵다.** PRD를 완성해도 TRD로 넘어가려면 새 탭을 열고, 내용을 복사하고, 다시 붙여넣는 수작업이 필요하다. 이 과정에서 컨텍스트가 소실되고, 귀찮음에 워크플로우가 중단된다.

### 문제의 증거

OST 분석에서 **OPP-2(단계 간 수동 복사·붙여넣기, Opportunity Score 0.86)**가 가장 높은 점수를 기록했다. 현재 어떤 핸드오프 메커니즘도 존재하지 않는다. 사용자는 새 탭을 열고, 날 텍스트를 복사하고, 수동으로 붙여넣는다.

**OPP-1(순서 불명확, Score 0.72)**과 **OPP-4(탭 닫을 때 작업 소실, Score 0.68)**도 같은 세션 이탈 패턴을 가중시킨다.

### 왜 지금인가

바이브 코딩 현상이 한국에서 빠르게 확산되고 있다. Cursor 한국 사용자가 2024년 기준 15만 명 이상으로 추산되며, "Cursor를 시작하기 전에 무엇을 준비해야 하는가"에 대한 수요가 함께 증가하고 있다. 경쟁사 중 한국어 우선, 순차 워크플로우를 함께 제공하는 곳은 없다. 이 공백을 선점할 시간이 있다.

## 4. 목표 및 성공 지표 (Goals & Success Metrics)

### OKR (90일 기준)

**Objective**: 사용자가 단일 세션에서 전체 기획 워크플로우를 완성할 수 있는 제품으로 전환한다.

| Key Result | 기준선 (현재) | 목표 (90일) | 측정 방법 |
|-----------|-------------|------------|---------|
| KR1: 풀체인 워크플로우 완료율 (PRD → TRD → IA → Use Cases → Design 모두 클릭) | ~0% (미측정) | 30% | GA4 이벤트: 모든 생성기 "생성/복사" 클릭 in 1 session |
| KR2: PRD 생성 후 IA로의 연속 이동율 | ~0% (핸드오프 없음) | 25% 이상 | "다음 단계로 보내기" 버튼 클릭률 |
| KR3: 주간 활성 프롬프트 생성자 (WAPG) | 기준치 설정 예정 | 기준치 대비 +40% | GA4 weekly unique "generate" event |
| KR4: 폼 재방문 시 자동 복원 후 제출 완료율 | 미측정 | 새 세션 대비 +40% | localStorage 복원 감지 후 submit 이벤트 비교 |

### North Star Metric

**주간 활성 프롬프트 생성자 (Weekly Active Prompt Generators, WAPG)** — 일주일에 한 번 이상 프롬프트를 생성한 고유 사용자 수.

이 지표는 단순 방문이 아니라 실제 가치를 경험한 사용자를 반영하며, 광고 수익과 미래 구독 전환율 양쪽 모두와 직접 연동된다.

## 6. 솔루션 및 기능 요건 (Solution & Feature Requirements)

### 핵심 솔루션 방향

OST 분석의 상위 3개 기회(OPP-2, OPP-1, OPP-4)에 대응하는 세 가지 솔루션을 순서대로 구현한다.

---

### 기능 1: 단계 간 컨텍스트 핸드오프 (OPP-2 대응) — 최우선

**기능 설명**

결과 다이얼로그에 "다음 단계로 보내기" 버튼을 추가한다. PRD 완료 후 클릭하면 생성된 PRD 내용을 sessionStorage에 저장하고 `/ia` 페이지로 이동한다. IA 페이지는 "PRD 컨텍스트 불러옴" 배너를 표시하고 컨텍스트 텍스트 영역을 자동으로 채운다.

**구현 범위**

- PRD → IA 핸드오프 버튼 (EXP-1: 가장 먼저 검증할 실험)
- IA → Use Cases 핸드오프 버튼
- Use Cases → Design 핸드오프 버튼
- TRD는 PRD와 병렬 트랙으로 분리 (독립적으로 사용 가능)
- 각 페이지에 "컨텍스트 불러옴" 배너 (닫기 버튼 포함)

**기술 접근**

- sessionStorage 키 규칙: `bji_handoff_{source}` (예: `bji_handoff_prd`)
- 목적지 페이지 mount 시 sessionStorage 확인 → 컨텍스트 텍스트 영역 자동 주입
- 이전 핸드오프 데이터가 있을 때만 배너 표시
- 추가 백엔드 없음, Zustand 상태와 sessionStorage만 사용

**수용 기준**

- PRD 결과 다이얼로그에서 "IA로 계속하기" 버튼 클릭 시 `/ia`로 이동하고 PRD 내용이 채워져 있다.
- 탭을 새로 열거나 브라우저를 닫으면 sessionStorage가 초기화된다 (프라이버시 보호).
- 사용자가 직접 `/ia`로 접근하면 배너 없이 빈 폼이 표시된다.

---

### 기능 2: 순차 워크플로우 홈페이지 다이어그램 (OPP-1 대응)

**기능 설명**

홈페이지의 무순서 툴 카드를 순차 워크플로우 섹션으로 교체한다. "Step 1: PRD → Step 2: TRD → Step 3: IA → Step 4: Use Cases → Step 5: Design"을 시각적 플로우차트로 표현하고, 각 단계에 직접 진입 CTA를 배치한다.

**구현 범위**

- 홈페이지 ToolsSection 컴포넌트 재설계 (EXP-2)
- 번호가 붙은 순차 카드 레이아웃 (수평 화살표 포함)
- TRD는 "선택적 병렬 트랙"으로 시각적으로 분리 표시
- 각 카드에 단계 설명 1줄 (예: "PRD: 제품 요구사항 정의") 포함

**수용 기준**

- 홈페이지 방문자가 5초 이내에 어떤 순서로 도구를 사용해야 하는지 파악할 수 있다.
- 각 단계 카드 클릭 시 해당 생성기 페이지로 이동한다.

---

### 기능 3: 폼 상태 자동 저장 (OPP-4 대응)

**기능 설명**

각 생성기 폼의 모든 필드 값을 onChange마다 localStorage에 자동 저장한다. 페이지 재방문 또는 새로고침 시 마지막 저장 상태를 자동 복원한다.

**구현 범위**

- localStorage 키 규칙: `bji_{generator}_draft` (예: `bji_prd_draft`)
- 모든 5개 생성기 폼에 적용
- "처음부터 시작" 버튼으로 저장된 상태 초기화 기능 필수 포함 (EXP-3에서 PRD 먼저 적용)
- 복원 시 상단에 "이전 작업을 불러왔습니다" 토스트 표시

**수용 기준**

- 폼 작성 중 탭을 닫고 재방문하면 마지막 입력 상태가 복원된다.
- "처음부터 시작" 클릭 시 폼이 초기화되고 localStorage 항목이 삭제된다.

---

### 기능 4: Use Cases 폼 구조화 (기술 부채 해소)

**기능 설명**

현재 UsecaseForm은 구조화된 입력 필드가 전혀 없고 정적 프롬프트만 생성한다. 이는 가장 낮은 품질의 생성기다. 다른 생성기와 동일한 수준의 폼 입력 구조를 추가한다.

**구현 범위**

- 최소 입력 필드: 시스템/서비스명, 주요 액터(Actor), 핵심 기능 목록 (3–5개), 비기능 요건 (선택)
- 기존 정적 프롬프트를 입력값 기반 동적 프롬프트로 교체
- IA 핸드오프에서 컨텍스트를 받을 수 있는 컨텍스트 텍스트 영역 추가

**수용 기준**

- UsecaseForm에 최소 3개 이상의 구조화된 입력 필드가 있다.
- 입력 필드가 비어 있으면 생성 버튼이 비활성화된다.

---

### 기술 가정 및 UX 고려사항

| 가정 | 리스크 | 완화 방안 |
|------|--------|---------|
| 사용자가 PRD → IA → Use Cases → Design 순서를 따른다 | 사용자가 순서 없이 사용하거나 여러 프로젝트를 동시에 진행할 수 있음 | 핸드오프는 덮어쓰기 방지 — 배너에서 "현재 내용 유지" 선택 가능 |
| sessionStorage가 한 세션 내에서 충분한 컨텍스트를 보존한다 | 매우 긴 프롬프트는 sessionStorage 한계에 도달할 수 있음 | 저장 전 5,000자 트런케이션 + 사용자 알림 |
| localStorage 자동 저장이 다른 프로젝트 작업을 방해하지 않는다 | 새 프로젝트 시작 시 이전 프로젝트 데이터가 남아 있음 | "처음부터 시작" 버튼 눈에 잘 띄게 배치 |

## 7. GTM 전략 (Go-to-Market Strategy)

### 포지셔닝

> "Cursor를 시작하기 전에 거치는 워크플로우 — 한국어로, 무료로, PRD부터 디자인 가이드까지 한 번에."

이 문장은 세 가지 경쟁 우위를 동시에 전달한다: 한국어 우선, 무료 진입, 순차 워크플로우 커버리지.

### 채널 전략

| 채널 | 목적 | 구체적 실행 | 우선순위 |
|------|------|------------|---------|
| **SEO (롱테일 한국어)** | 유기 유입 기반 구축 | "PRD 작성 방법 AI", "Cursor 프롬프트 기획", "바이브 코딩 문서화" 랜딩 페이지 | 1순위 |
| **Disquiet.io** | 바이브 코더 커뮤니티 직접 접근 | "이 도구로 PRD 5분 만에 만들었습니다" 실사용 후기 포스트 | 1순위 |
| **X(트위터) 한국 개발자** | 바이럴 루프 트리거 | 5단계 워크플로우 스크린샷 스레드, "#바이브코딩" 해시태그 활용 | 2순위 |
| **YouTube / Velog 튜토리얼** | 신뢰 구축 및 사용법 교육 | "5분 PRD 완성 챌린지" 시리즈 (화면 녹화 + 실제 Cursor 사용 연동 시연) | 2순위 |
| **FastCampus 파트너십** | 기관 채널 유입 | AI 개발 강의 수업 자료에 NextCursor 포함 제안 | 3순위 |
| **제품 주도 성장 (PLG)** | 자연스러운 공유 유도 | 생성된 문서에 "NextCursor로 생성됨" 푸터 + 워터마크 (클릭 시 도구로 연결) | 2순위 |

### 메시지 프레임워크

**핵심 가치 제안:**
"아이디어만 있으면 됩니다. AI 기획 문서는 NextCursor가 만들어 줍니다."

**세그먼트별 변형:**

- 바이브 코더에게: "코딩 전 30분을 아끼세요. Cursor에 넣을 PRD, IA, 디자인 가이드를 한 번에."
- 주니어 PM에게: "시니어 PM이 쓴 것처럼 보이는 한국어 TRD. 템플릿 찾느라 시간 낭비 끝."
- 프론트엔드 개발자에게: "Tailwind 준비된 색상 팔레트와 컴포넌트 스펙. IA와 Design 문서가 같은 언어를 씁니다."

### 성공 지표 (GTM)

| 단계 | 지표 | 목표 (90일) |
|------|------|------------|
| 인지 (Awareness) | 목표 키워드 검색 노출 | 주요 5개 키워드 상위 30위 |
| 방문 (Engagement) | 월간 활성 사용자 (MAU) | 3,000 MAU |
| 활성화 (Activation) | 폼 완성 후 프롬프트 생성율 | 방문자의 25% |
| 유지 (Retention) | 30일 재방문율 | 20% |
| 수익 (Revenue) | 월간 AdSense 수익 | ₩50,000 (기준점 설정) |

### 출시 계획

**Pre-Launch (Day 1–14): 기반 구축**
- GA4 이벤트 계측 설정 (generate, copy, handoff_click)
- "다음 단계로 보내기" 버튼 (EXP-1) 구현 및 A/B 태그 추가
- 홈페이지 워크플로우 다이어그램 (EXP-2) 배포
- SEO 랜딩 페이지 초안 작성

**Launch Day (Day 15): 커뮤니티 발표**
- Disquiet.io 신제품 소개 포스트 (실사용 GIF 포함)
- X(트위터) 스레드: "5단계 워크플로우로 PRD 만드는 법"
- 기존 방문자 대상 이메일/뉴스레터 발송 (구독자가 있는 경우)

**Post-Launch (Day 16–90): 학습과 반복**
- 매주 GA4 데이터 리뷰 (핸드오프 버튼 클릭률, 체인 완료율)
- Day 30: EXP-1 결과 평가 → IA → Use Cases 핸드오프 확장 여부 결정
- Day 45: 얼리어답터 인터뷰 5회 실시
- Day 60: localStorage 자동 저장 (EXP-3) 배포
- Day 90: 목표 KR 달성 여부 점검, v2 범위 결정

### 90일 로드맵

```
Week 1–2    기반 구축
            ├── GA4 이벤트 계측 설정

            ├── YouTube 튜토리얼 제작
```

## 8. 리스크 및 가정 (Risks & Assumptions)

### 주요 가정

| # | 가정 | 리스크 수준 | 검증 방법 | 검증 시점 |
|---|------|:----------:|---------|---------|
| 1 | 한국 개발자/바이브 코더가 한국어 AI 기획 프롬프트를 적극적으로 검색한다 | 높음 | Search Console 키워드 데이터, 2주 SEO 실험 | Day 30 |
| 2 | 사용자가 PRD → IA → Use Cases → Design 순서를 대체로 따른다 | 중간 | 세션 내 생성기 간 이동 패턴 분석 | Day 30 |
| 3 | "다음 단계로 보내기" 버튼이 수동 복사·붙여넣기보다 유의미하게 높은 클릭률을 기록한다 | 중간 | EXP-1 클릭률 측정 (목표: 기준선 대비 +25%) | Day 14 |
| 4 | 바이브 코더는 API 키 없이 프롬프트 복사·붙여넣기로 만족한다 | 높음 | "AI 생성" vs "복사" 버튼 클릭 비율 추적 | Day 30 |
| 5 | ₩9,900/월 프리미엄 구독에 대한 지불 의향이 존재한다 | 높음 | 프라이싱 페이지 웨이팅 리스트 전환율 측정 | Day 90 |
| 6 | 순차 워크플로우 완성 경험이 재방문율을 높인다 | 중간 | 풀체인 사용자 vs 단일 도구 사용자 30일 재방문율 비교 | Day 60 |

### 주요 리스크

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 |
|--------|:----------:|:------:|---------|
| Cursor가 한국어 PRD 생성 기능을 네이티브로 추가 | 중간 | 높음 | "Cursor 전 워크플로우"로 포지셔닝을 더 명확히; Cursor와의 통합(cursorrules 내보내기)으로 공존 전략 |
| SEO 트래픽 확보 실패 (경쟁 키워드 포화) | 중간 | 중간 | 커뮤니티 직접 배포(Disquiet, X)로 SEO 의존도 분산 |
| 가정 #2 실패: 사용자가 순서 없이 사용 | 중간 | 중간 | 핸드오프 버튼이 없어도 각 생성기가 독립적으로 가치를 제공하도록 유지 |
| Use Cases 폼 구조화 후 기존 사용자 혼란 | 낮음 | 낮음 | 점진적 출시, 기존 정적 프롬프트를 "빠른 시작" 옵션으로 병존 |
| AdSense 수익이 개발 비용을 정당화하지 못함 | 중간 | 중간 | 프리미엄 구독 웨이팅 리스트를 Day 90 전에 런칭하여 수익 다각화 준비 |

### 현재 미결 문제

- **애널리틱스 미구축**: GA4 이벤트 계측이 없어 현재 기준선 데이터가 없다. EXP-1 실행 전 최우선 해결 필요.
- **Use Cases 폼 설계**: 현재 입력 필드가 0개다. 어떤 구조화된 입력이 최고의 Use Cases 프롬프트를 만드는지는 아직 검증되지 않았다.
- **프리미엄 전환 경로**: ₩9,900/월 구독 모델로의 전환 UX(페이월 위치, 무료 한도)는 이 PRD 범위 밖이며 별도 기획이 필요하다.
